## QC Notes

All 44 ACs confirmed via independent code review with file:line evidence. No overrides needed. Implementation matches plan v5 spec.

Key observations from QC review:
- **Config-driven architecture**: Both `computeTicketPageConfig` (SupportPageClient.tsx:50-76) and `computePlanCardConfig` (PlanPageClient.tsx:47-95) serve as single sources of truth, eliminating scattered conditional logic.
- **Dual credit pools**: `formatCredits` (PlanPageClient.tsx:101-112) correctly handles 3 display scenarios: plan+purchased, purchased-only, plan-only.
- **Legal module**: lib/legal.ts provides fetchLegalDoc (GET, public, 24h cache) and acceptLegalDocs (POST, Bearer auth) — clean separation from actions.ts server action layer.
- **No hardcoded pricing**: All plan values flow from `plan.details.*` (PlanDetailClient) and `license.alaCarte[]` (all pages). Exception: toast "48 hours" at SupportPageClient.tsx:168 is UI copy, not contractual SLA — acceptable per plan spec.
- **ManagePageClient deletion**: All functionality preserved in TermsPageClient's LicenseKeyTab — same activateLicense/refreshLicense actions, same SettingsField telemetry toggle, same key masking pattern.
